# GitHub更新手順

## 方法1: VS CodeのGit機能を使用（推奨）

1. **VS Codeでソース管理パネルを開く**
   - 左側のソース管理アイコン（分岐マーク）をクリック
   - または `Ctrl + Shift + G`

2. **変更をステージング**
   - 「変更」セクションに変更されたファイルが表示されます
   - 各ファイルの横の「+」をクリックしてステージング
   - または「変更」の横の「+」をクリックしてすべてをステージング

3. **コミット（日本語メッセージ）**
   - 上部のメッセージボックスにコミットメッセージを日本語で入力
   - 例: `UI: 全体的なデザインをモダンでスタイリッシュに改善`
   - 例: `fix: map.htmlの表示問題を修正`
   - 例: `feat: レスポンシブデザインに対応`
   - 「✓」ボタンをクリックしてコミット
   - **注意**: VS Codeの設定で日本語コミットメッセージが有効になっています

4. **プッシュ**
   - 左下の「同期」アイコンをクリック
   - または「...」メニューから「プッシュ」を選択

### コミットメッセージの形式（推奨）

変更の種類をプレフィックスとして使用すると分かりやすくなります：

- `feat:` - 新機能の追加
- `fix:` - バグ修正
- `ui:` - UI/UXの改善
- `style:` - コードスタイルの変更（動作に影響なし）
- `refactor:` - リファクタリング
- `docs:` - ドキュメントの更新
- `perf:` - パフォーマンス改善
- `test:` - テストの追加・修正
- `chore:` - その他の変更（ビルド設定など）

## 方法2: GitHub Desktopを使用

1. **GitHub Desktopを開く**
2. **変更を確認**
   - 左側に変更されたファイルが表示されます
3. **コミット**
   - 下部の「Summary」にコミットメッセージを入力
   - 「Commit to main」をクリック
4. **プッシュ**
   - 「Push origin」をクリック

## 方法3: Git Bashを使用

Git Bashがインストールされている場合：

```bash
git add .
git commit -m "feat: ログイン機能の改善と管理者ページへの自動遷移を追加"
git push
```

**日本語コミットメッセージの設定**
- プロジェクトには`.gitmessage`テンプレートが設定されています
- テンプレートを使用する場合: `git commit`（エディタが開きます）
- 直接メッセージを指定する場合: `git commit -m "メッセージ"`
