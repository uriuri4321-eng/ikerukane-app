# イケルカネ - 卒論中間発表 発表内容案

## 1. タイトルスライド（30秒）
- **タイトル**: イケルカネ - お金の抑止力で遅刻をさせない
- **副題**: 位置情報を活用した遅刻防止アプリケーション
- **発表者名・所属**

## 2. 研究背景・問題提起（1分）
### 遅刻問題の現状
- 約束の時間に遅れることは、信頼関係に影響を与える
- 自己管理が難しい場面がある
- 単なる「意識改革」だけでは不十分な場合がある

### 解決アプローチ
- **経済的インセンティブ**を活用
- 遅刻すると**お金がかかる**仕組み
- 位置情報技術を活用した**自動検知システム**

## 3. システム概要（1.5分）
### コンセプト
「お金の抑止力で遅刻をさせない」

### 主な機能
1. **予定管理機能**
   - カレンダーで予定を登録
   - 目的地と期日を設定

2. **位置情報追跡機能**
   - リアルタイムで現在位置を取得
   - 目的地までの距離を計算
   - 到着予測時間を表示

3. **課金システム**
   - 遅刻した場合、設定した金額が課金される
   - 到着した場合、課金を阻止（お金はかからない）

4. **統計・管理機能**
   - 個人の遅刻履歴を確認
   - 管理者ダッシュボードで全体統計を確認

## 4. 技術スタック（1分）
### フロントエンド
- HTML5 / CSS3 / JavaScript（Vanilla JS）
- レスポンシブデザイン（スマートフォン対応）

### バックエンド・データベース
- **Firebase Authentication**: ユーザー認証（Google Sign-In対応）
- **Firebase Firestore**: クラウドデータベース
  - 予定データ、位置情報履歴、課金情報を保存
  - 複数デバイス間でのデータ同期

### 外部API
- **Google Maps JavaScript API**: 地図表示、位置検索、距離計算

### 特徴
- サーバーレスアーキテクチャ
- リアルタイムデータ同期
- オフライン対応（localStorageをキャッシュとして使用）

## 5. システムの動作フロー（1.5分）
### ユーザー側の流れ
1. **予定登録**: カレンダーで予定を作成
2. **目的地設定**: 地図上で目的地と課金額を設定
3. **当日**: アプリを開くと自動的に位置情報を追跡開始
4. **到着確認**: 目的地に到着したかどうかを自動判定
5. **結果**: 
   - 到着成功 → 課金なし
   - 遅刻 → 課金額が記録される

### 技術的な処理
- 位置情報の取得: `navigator.geolocation.watchPosition()`
- 距離計算: Haversine formula（球面距離計算）
- リアルタイム更新: 定期的な位置情報取得と距離チェック

## 6. 実装のポイント・工夫（1.5分）
### データ移行戦略
- 既存のlocalStorageデータをFirestoreに自動移行
- オフライン時もlocalStorageで動作可能

### ユーザビリティ

### セキュリティ

- 管理者権限の実装
